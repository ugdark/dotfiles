# 作業の再開

中断されたセッションから作業を再開する。

## 使用場面

Claudeは以下の状況で**自動的にこのスキルを適用**する：

- 「作業を再開したい」「続きをやりたい」と言われた時

また、`/d-quest-resume` で手動呼び出しも可能。

## 実行フロー

### Step 1: quest確認

- Glob: `~/.dotfiles/vault/quests/active/*.md` でactive questを取得
- **1件のみの場合**: 自動的にそのquestを選択（ユーザーに聞かない）
- **2件以上の場合**: AskUserQuestion で一覧を表示し、ユーザーに選択を聞く

### Step 2: questファイルの読み込み

- Read: 選択されたquest.md

### Step 3: 対応状況の分析

- 「## 対応状況」セクションを解析
- `[x]` 完了済みタスクをカウント
- `[ ]` 未完了タスクをカウント

### Step 4: 実装メモの確認

- 「## 実装メモ」セクションを確認
- 最後の「の結果」セクションを探す
- 実装完了日、作成ファイル、変更点を取得

### Step 5: 状況報告

以下の形式でユーザーに報告する：

```
現在の状況を確認しました。

【Plan】: [questファイル名から抽出したタイトル]
【進捗】: [完了数]/[全体数] 完了

【完了済み】:
- [x] [タスク1]
- [x] [タスク2]

【残りタスク】:
- [ ] [タスクA]
- [ ] [タスクB]

【最後の実装】:
[タスク名]（[完了日]完了）
- [作成ファイル1]
- [作成ファイル2]

【次のタスク候補】:
[最初の未完了タスク]

次はどのタスクに進みますか？
```

### Step 6: ユーザー指示待ち

- ユーザーの指示を待つ
- 「次に進む」 → `/d-quest-implement` フローを開始
- 「別のタスク」 → 指定タスクで実装を開始
- 「一旦停止」 → 停止して待機

## やり取り履歴の追記（必須）

このスキル実行中の **質問・回答・方針判断** は、questファイル末尾の `## やり取り履歴` セクションの **最終行に逐次追記** する。

- 追記先: `## やり取り履歴` セクション末尾（既存記述の下に追加）
- フォーマット例:
  ```markdown
  ### YYYY-MM-DD HH:mm （d-quest-resume）
  - 再開状況: [完了数]/[全体数]
  - 次タスク: 〜
  - メモ: 〜
  ```
- セクションが存在しない場合のみ、ファイル末尾に新規作成して追加する
