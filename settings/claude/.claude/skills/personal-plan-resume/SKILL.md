# 作業の再開

中断されたセッションから作業を再開する。

## 使用場面

Claudeは以下の状況で**自動的にこのスキルを適用**する：

- 「作業を再開したい」「続きをやりたい」と言われた時

また、`/personal-plan-resume` で手動呼び出しも可能。

## 実行フロー

### Step 1: plan確認

- ユーザーに「どのplanで作業していましたか？」と確認
- または active/ のplan一覧を表示
  - Glob: `~/.dotfiles/local/plans/active/*.md`

### Step 2: planファイルの読み込み

- Read: ユーザーが指定したplan.md

### Step 3: 対応状況の分析

- 「## 対応状況」セクションを解析
- `[x]` 完了済みタスクをカウント
- `[ ]` 未完了タスクをカウント

### Step 4: 実装メモの確認

- 「## 実装メモ」セクションを確認
- 最後の「の結果」セクションを探す
- 実装完了日、作成ファイル、変更点を取得

### Step 5: 状況報告

以下の形式でユーザーに報告する：

```
現在の状況を確認しました。

【Plan】: [planファイル名から抽出したタイトル]
【進捗】: [完了数]/[全体数] 完了

【完了済み】:
- [x] [タスク1]
- [x] [タスク2]

【残りタスク】:
- [ ] [タスクA]
- [ ] [タスクB]

【最後の実装】:
[タスク名]（[完了日]完了）
- [作成ファイル1]
- [作成ファイル2]

【次のタスク候補】:
[最初の未完了タスク]

次はどのタスクに進みますか？
```

### Step 6: ユーザー指示待ち

- ユーザーの指示を待つ
- 「次に進む」 → `/personal-plan-implement` フローを開始
- 「別のタスク」 → 指定タスクで実装を開始
- 「一旦停止」 → 停止して待機
