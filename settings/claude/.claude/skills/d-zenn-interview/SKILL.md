# Zenn記事ヒアリング

記事ネタを質問で深掘りし、関連素材を集め、`drafts/<slug>.md` に **公開できる水準の本文ドラフト** を書き出す。
これは Zennワークフローの **②ヒアリング＆執筆** 段階。`d-zenn-ideas`（①案出し）から繋がり、完成したドラフトは `d-zenn-post`（③投稿）で公開する。

## 使用場面

- 「Zennのネタを深掘りして」「ヒアリングして」「記事の下書き作って」と言われた時
- `d-zenn-ideas` で候補を選んだ後
- `/d-zenn-interview [テーマ]` で手動呼び出し

## 実行フロー

### Step 1: テーマの確定

- 引数や直前の会話（`d-zenn-ideas` の選定結果）からテーマが分かればそれを使う
- 不明なら「どのネタを記事にしますか？」と確認する

### Step 2: 素材の収集

記事の裏付けになる既存素材を集める（Bash の `find` / `grep` で走査。Globは使えない）:

- `~/.dotfiles/vault/knowledge-base/` — 関連ナレッジ
- `~/.dotfiles/vault/daily/` — 関連する出来事・学び
- `~/.dotfiles/vault/quests/` — 関連する作業記録
- プロジェクトのドキュメント（ユーザーが場所を示せばそこも）— README・docs・ADR・ログ等
- 既存Zenn記事（重複回避）

見つかった素材は Read で内容を確認し、本文の根拠・コマンド例・ハマり事例として活用する。

### Step 3: ヒアリング（深掘り質問）

本文を書くために不足している情報を、対話で深掘りする。`AskUserQuestion` も活用しつつ、以下の観点を埋める:

1. **想定読者**: 誰が読む？どんなレベル？
2. **読者の課題 / Before**: 読む前にどんな悩み・状態か
3. **記事で得られるもの / After**: 読んだ後どうなれるか（記事の主張・結論）
4. **核となるポイント**: 伝えたい要点を3〜5個
5. **具体例 / コード / 図**: 載せる実例・コードスニペット・スクショの有無
6. **type と topics**: tech/idea、トピック（最大5個・英小文字推奨）
7. **タイトル案・emoji**

一度に全部聞かず、要点を絞って数回に分けて聞く。素材（Step 2）で埋まる項目は質問しない。回答が薄い箇所は掘り下げる。

### Step 4: slug を決める

- 半角英小文字・数字・ハイフン・アンダースコア、12〜50文字
- `drafts/` `articles/` に同名がないか `find` で確認

### Step 5: 本文ドラフトを drafts/ に書き出す

Read で `~/.dotfiles/vault/articles/zenn/drafts/_template.md` を確認し、Write で `drafts/<slug>.md` を作成する。**メモや骨子ではなく、公開できる水準の本文まで書く**:

```markdown
---
title: "<タイトル>"
emoji: "<絵文字>"
type: "tech"        # tech | idea
topics: [<トピック>]  # 最大5個
published: false
---

:::message
本記事は Claude に下書きを生成させ、筆者が事実確認・構成調整・加筆修正を行ったものです。
:::

## はじめに
<本文（課題提起・記事のゴール）>

## <核ポイントの見出し>
<本文。コード例・コマンド・表などを含める>

...

## まとめ
<本文（主張の再確認・次アクション）>
```

執筆方針:
- Step 2 の素材・Step 3 の回答を根拠に、各セクションの**本文を実際に書く**
- コードブロックには言語指定を付ける。Zenn記法（`:::message`, `:::details`, URL単独行のカード）を活用
- 事実確認が必要な箇所（料金・バージョン・最新仕様など）は、断定せず HTMLコメント `<!-- 要確認: ... -->` を残す
- AI下書き注記（`:::message`）を冒頭に置く（既存記事と統一）

### Step 6: 完了報告と次アクション

- 作成した `drafts/<slug>.md` のパスと、見出し構成・要確認メモを提示
- 「プレビュー・公開は `d-zenn-post` で進められます」と案内する

## 重要ルール

- **成果物は drafts/ の本文ドラフト**: 骨子止まりにせず、公開水準の本文まで書く。推敲・プレビュー・公開は `d-zenn-post` に渡す
- **素材を裏付けに使う**: 憶測で書かず、knowledge-base・daily・プロジェクトdocs等の実素材に基づく
- **要確認箇所はコメントで明示**: 料金・バージョン等の不確実な事実は `<!-- 要確認 -->` を残し、断定しない
- **published は false**: 下書き段階なので必ず false
- **AI下書き注記を入れる**: `:::message` を冒頭に置く
- **slugはZenn制約を守る**: 半角英小文字・数字・ハイフン・アンダースコア、12〜50文字
- **次段階へ繋ぐ**: ドラフト完成後は `d-zenn-post` へ渡す導線を示す
