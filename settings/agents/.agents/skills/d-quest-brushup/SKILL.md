# ブラッシュアップ

計画・要件・設計を、質問で深掘りして磨き直す。曖昧なまま実装に入って手戻りするのを防ぐ。主な対象は quest だが、quest に限らず壁打ちしたいテーマ全般に使える。

## 使用場面

`/d-quest-brushup` で手動呼び出し。「磨いて」「要件を深掘りして」と言われた時にも適用する。

## 実行フロー

### Step 1: 対象の特定

- 引数があればそれを対象にする
- quest を磨く場合、引数がなければ Glob `~/.desk/quests/active/*.md`（1件 → 自動選択 / 複数 → AskUserQuestion で選択）
- quest 以外（会話中の計画・要件メモ等）が対象なら、それを1文で言語化して確認する

### Step 2: 現状把握

- 対象を読む（quest なら 要望・要件 / 前提条件 / 対応状況 / 概要 / 実装メモ）
- **関連するコードベースを先に調べる**。調べれば分かることは質問しない

### Step 3: 質問で深掘り（コア）

共通認識に達するまで、**1問ずつ**詰める。

- 1ターンの質問は1つだけ。可能なら `AskUserQuestion` で選択肢＋推奨を提示する
- 「幅」より「深さ」。1つのテーマを掘り尽くしてから次へ進む
- 目的・スコープ・前提・成功条件・代替案・リスクのうち、曖昧な点を突く
- 忖度せず、矛盾や抜けを率直に指摘する

### Step 4: 反映

詰めた結果を対象に反映する。

- **quest が対象**: 要望・要件 / 前提条件 / 概要 / 完了条件 / 対応状況 を更新（テストは各実装タスクに内包）
- **quest 以外**: 簡潔な計画（目的 / スコープ / 前提 / 方針 / 成功条件）にまとめて返す

### Step 5: やり取り履歴の追記（questが対象の場合・必須）

quest末尾の `## やり取り履歴` セクション末尾に逐次追記する。

```markdown
### YYYY-MM-DD HH:mm （d-quest-brushup）
- Q: 〜
- A: 〜
- メモ: 〜
```

## 重要ルール

- 質問は1問ずつ。一度に多く聞かない
- 共通認識に達するまで安易に切り上げない
- questは必ず日本語で記載・簡潔に
- 反映先・粒度に迷う場合はユーザーに1つ確認してから書く
