# 軽いメモ追記

短い相談・調査・Q&Aなど「questにするほどじゃないけど忘れたくない話」を、`~/.dotfiles/vault/note.md` の末尾に追記する。

## 使用場面

Claudeは以下の状況で**自動的にこのスキルを適用**する：

- 「メモして」「note追加」「これメモっといて」「note.mdに残して」と言われた時

また、`/d-note` で手動呼び出しも可能。引数で本文を直接渡すこともできる。

## 実行フロー

### Step 1: 追記内容の決定

- **引数あり**: 引数の内容をそのまま追記対象にする
- **引数なし**: 直前の会話の要点を 3〜10 行程度でサマリする
  - 質問されたテーマ
  - 調べて分かった結論・要点
  - 参考になりそうなコマンド/コード片があれば最小限だけ

### Step 2: note.md の存在確認・作成

- `~/.dotfiles/vault/note.md` が存在するか確認
  - Bash: `ls ~/.dotfiles/vault/note.md 2>/dev/null`
- 存在しない場合のみ Write で新規作成（先頭に `# Note` だけ書く）

### Step 3: 末尾追記

- Edit (or Read → Edit) で **ファイル末尾** に以下のフォーマットで追記
- **必ず末尾追加・先頭挿入や中間挿入は禁止**

```markdown

## YYYY-MM-DD HH:mm - [一言タイトル]

[本文：要点・結論を箇条書き or 短文で。冗長にしない]
```

- 日付は今日の日付、時刻は現在時刻
- タイトルは内容を15文字程度で要約（例: `Scalaのblocking`、`IDEAの除外フォルダ恒久化`）

### Step 4: 完了報告

- 追記したタイトルとファイルパスを1行で報告
- 例: `note.md に追記しました: ## 2026-04-28 15:30 - Scalaのblocking`

## 重要ルール

- ファイルは `~/.dotfiles/vault/note.md` のみ。日付別ファイルは作らない
- **常に末尾追記**。既存内容の編集・削除はしない
- 1エントリは簡潔に（10行以内が目安）
- コードブロックを含める場合も最小限（フルコードは貼らない、要点のみ）
- アーカイブはしない（ずっと1ファイルに溜める運用）
